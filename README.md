Junior Angular Developer --- DevHive

Репозиторій для роботи:\

Кандидату надається існуючий проєкт на Angular 11. Проєкт є робочим та
містить базову логіку.\
Завдання --- модернізувати його відповідно до сучасних підходів Angular
та побудувати коректну архітектуру.

## Мета завдання

-   Оцінити глибину знань Angular
-   Перевірити архітектурне мислення
-   Перевірити вміння працювати з Nx
-   Оцінити якість Git-історії
-   Перевірити навички роботи з динамічними структурами

------------------------------------------------------------------------

## 1. Оновлення Angular

-   Оновити проєкт з Angular 11 до Angular 20
-   Забезпечити стабільну роботу застосунку після оновлення
-   За потреби виконати рефакторинг із використанням сучасних
    можливостей фреймворка

------------------------------------------------------------------------

## 2. Міграція на Nx

-   Перевести проєкт на Nx workspace
-   Поточний застосунок повинен залишитися в папці `apps`
-   Побудувати логічну структуру бібліотек

------------------------------------------------------------------------

## 3. Створення пакету `weather`

Створити бібліотеку `weather` з двома secondary entry points:

-   `weather/state`
-   `weather/ui`

### weather/state

-   Реалізувати state management на Angular Signals
-   Інтегрувати будь-яке OpenAPI з погодою
-   Логіка повинна бути повністю відокремлена від UI

### weather/ui

-   Використати динамічний block builder
-   UI повинен взаємодіяти зі state через `weather/state`

------------------------------------------------------------------------

## 4. Створення пакету `block-builder`

Створити бібліотеку `block-builder`.

### Функціональні вимоги

Білдер повинен:

-   Динамічно створювати блоки
-   Працювати виключно з плоским об'єктом (без вкладених структур)
-   Підтримувати зміну порядку блоків

Кожен блок повинен містити:

-   Select з ключами
-   Відображення значення відповідного ключа
-   Можливість змінити ключ, після чого значення повинно оновлюватися
-   Кнопку видалення

В кінці списку блоків:

-   Кнопка додавання нового блоку
-   Новий блок повинен працювати за тією ж логікою

------------------------------------------------------------------------

## 5. Git-вимоги

-   Зробити fork репозиторію
-   Вести чисту Git-історію
-   Один логічний крок --- один коміт
-   Створити окрему гілку: `epic/dynamic`
-   Налаштувати GitHub Action:
    -   Білд запускається вручну (не на кожен коміт)
    -   Перед здачею завдання повинен бути один успішний build

------------------------------------------------------------------------

## 6. Опціональне завдання (буде суттєвим плюсом)
